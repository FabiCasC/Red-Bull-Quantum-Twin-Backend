# Red Bull Quantum-Twin 2026 — Backend

FastAPI backend with ML pipeline for F1 digital twin predictions.

## Setup

### 1. Install
```bash
pip install -r requirements.txt
```

### 2. Get data
```bash
# Kaggle:
kaggle datasets download -d aadigupta1601/f1-strategy-dataset-pit-stop-prediction
unzip *.zip -d data/

# HuggingFace:
python data/export_huggingface.py
```

### 3. Train model (run once)
```bash
python ml/train_model.py
```

### 4. Start API
```bash
uvicorn main:app --reload --port 8000
```

### 5. Docs
http://localhost:8000/docs

## Architecture
Local: SQLite → Oracle Autonomous Database (OCI) in production
ML: Multiple Linear Regression (scikit-learn)
Data: Kaggle F1 Strategy + HuggingFace F1 Telemetry Montreal 2023

## Project Structure
```
backend/
├── data/
│   ├── f1_strategy.csv          ← from Kaggle
│   ├── f1_telemetry.csv         ← exported from HuggingFace
│   ├── f1_combined_clean.csv    ← generated by train_model.py
│   ├── results.csv              ← real vs predicted (frontend chart)
│   └── real_vs_predicted.png    ← model results plot
├── ml/
│   ├── train_model.py           ← full pipeline: load, clean, merge, train, save
│   ├── model.pkl                ← saved after training
│   ├── scaler.pkl               ← saved after training
│   └── metrics.json             ← MAE, MSE, R² saved as JSON
├── db/
│   └── quantum_twin.db          ← SQLite (Oracle Autonomous DB in production)
├── main.py                      ← FastAPI app
├── database.py                  ← SQLite setup
├── requirements.txt
└── README.md
```
